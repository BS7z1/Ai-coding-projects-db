## 1. 概述

### 1.1 背景、现状和业务目标

**背景**：
当前信贷业务中，借据信息分散记录于各部门，缺乏统一的台账管理平台，导致信息不一致、审批流程不规范、审计追溯困难。为满足监管合规要求及内部精细化管理需要，需建设统一的信贷借据台账管理与复核审批功能模块。

**现状**：
- 借据台账依赖 Excel 手工维护，易出错，版本混乱；
- 审批过程无电子留痕，无法满足监管审计要求；
- 绿色信贷、涉农信贷、担保信息等专项标签缺乏统一归口管理。

**业务目标**：
1. 建立标准化借据台账，支持申请岗对借据信息的增、删、改、查、批量导入、批量删除；
2. 建立两级审核机制：申请岗录入并提交，复核岗进行在线审批；
3. 全程电子化留痕，记录提交人、提交时间、复核人、复核时间及审批结论；
4. 支持绿色信贷、涉农信贷等专项分类标识，为监管报送提供数据基础。

### 1.2 术语定义

| 序号 | 术语名称 | 定义 |

## 3. 整体系统功能分布

### 3.1 【F01】借据台账管理（申请岗）

#### 3.1.1 特性条目及业务场景说明

申请岗通过本页面进行借据信息的全生命周期管理。业务场景包括：

- **场景 1：新增录入**：申请岗手动逐条录入新增借据，填写所有必填字段后保存为草稿，确认无误后提交复核。
- **场景 2：修改维护**：对草稿状态或被拒绝的借据进行字段修改，修改后可重新提交。
- **场景 3：删除**：对草稿状态的借据进行单条或批量删除，已提交/已通过的记录不可删除。
- **场景 4：批量导入**：通过下载标准模板，填写后上传 Excel 批量导入借据信息，系统逐行校验并给出导入结果报告。
- **场景 5：提交复核**：确认信息无误后逐条提交给复核岗，提交后状态变为"待复核"，不可再修改。
- **场景 6：查看审批结果**：查看复核岗的审批结论，若被拒绝可查看拒绝原因。

#### 3.1.2 业务流程

```
【新增/导入】→ 录入借据信息 → 保存（草稿）→ 确认信息 → 提交复核
                                    ↑
                    （被拒绝后可修改，重新提交）
```

#### 3.1.3 业务规则

| 序号 | 规则名称 | 规则内容 |
|---|---|---|
| BR-F01-01 | 借据号唯一性 | 同一系统内借据号不可重复，新增/导入时重复则报错 |
| BR-F01-02 | 企业统一信用代码格式 | 必须为 18 位字母数字组合，不满足格式则校验不通过 |
| BR-F01-03 | 担保机构必填条件 | "是否有担保"选择"是"时，"担保机构"字段必填 |
| BR-F01-04 | 日期逻辑校验 | 到期日期必须晚于放款日期，否则不允许保存 |
| BR-F01-05 | 利率范围限制 | 利率取值范围为 0.0001%～100%，超出范围则报错 |
| BR-F01-06 | 借据金额精度 | 借据金额单位为元，保留2位小数，最大值不超过 9,999,999,999.99 |
| BR-F01-07 | 状态限制修改 | 仅"草稿"和"已拒绝"状态的借据可修改；"待复核"和"已通过"状态不可编辑 |
| BR-F01-08 | 状态限制删除 | 仅"草稿"状态的借据可删除（包括批量删除）；其他状态禁止删除 |
| BR-F01-09 | 批量导入行数限制 | 单次批量导入最多支持 1000 条；超出则提示拆分导入 |
| BR-F01-10 | 批量导入错误处理 | 导入校验逐行进行，校验失败行给出错误说明，不影响其他行入库；最终展示成功/失败数量汇总 |
| BR-F01-11 | 已通过记录不可提交 | "已通过"状态记录不可再次提交复核 |

### 3.2 【F02】借据复核审批（复核岗）

#### 3.2.1 特性条目及业务场景说明

复核岗通过本页面对申请岗提交的借据台账进行审批。业务场景包括：

- **场景 1：查看待审批列表**：复核岗查看所有状态为"待复核"的借据记录列表；
- **场景 2：审批详情核查**：点击查看借据详情，核对所有字段信息，查看提交人信息；
- **场景 3：审批通过**：确认信息无误后，点击"通过"按钮，系统记录复核人、复核时间，状态变更为"已通过"；
- **场景 4：审批拒绝**：发现信息有误时，点击"拒绝"按钮，必须填写拒绝原因，系统记录相关信息，状态变更为"已拒绝"，同时通知申请岗修改；
- **场景 5：历史审批记录查询**：复核岗可查看已处理的历史审批记录（已通过/已拒绝），支持条件筛选。

#### 3.2.2 业务流程

```
复核岗进入审批页面
      ↓
查看待复核列表
      ↓
点击"审批"进入详情页
      ↓
核查借据信息（含提交人、提交时间）
      ↓
     选择审批结论
    ↙            ↘
  通过            拒绝（必填拒绝原因）
   ↓                  ↓
记录复核人+         记录复核人+
复核时间             复核时间+拒绝原因
   ↓                  ↓
状态→已通过        状态→已拒绝
   ↓                  ↓
通知申请岗         通知申请岗修改重提
```

#### 3.2.3 业务规则

| 序号 | 规则名称 | 规则内容 |
|---|---|---|
| BR-F02-01 | 审批操作权限 | 仅复核岗可进行审批操作，申请岗无权访问本页面 |
| BR-F02-02 | 拒绝原因必填 | 审批结论为"拒绝"时，拒绝原因不可为空，且不少于5个字 |
| BR-F02-03 | 审批记录不可撤销 | 审批通过或拒绝后，结论不可更改；如需调整须申请岗重新录入提交 |
| BR-F02-04 | 审批人限制 | 复核岗不可审批自己提交的借据（若复核岗兼有申请岗角色时适用） |
| BR-F02-05 | 系统自动记录时间 | 审批时间由系统自动生成，不允许人工填写或修改 |
| BR-F02-06 | 批量审批 | 支持在列表页勾选多条"待复核"记录进行批量通过（批量拒绝需逐条填写原因，不支持批量操作） |

#### 3.2.6 异常处理

| 序号 | 异常场景 | 处理方式 |
|---|---|---|
| 1 | 拒绝时未填写原因 | 提示"请填写拒绝原因后再提交" |
| 2 | 拒绝原因少于5字 | 提示"拒绝原因不能少于5个字" |
| 3 | 批量通过时含非待复核状态记录 | 提示"所选记录中含有非待复核状态的记录，请重新选择" |

| 5 | 审批操作超时（网络异常） | 提示"操作失败，请刷新页面后重试"，不允许重复提交 |

#### 3.2.7 其他

- **审批幂等保护**：防止因网络重复提交导致重复审批，后端需做幂等校验；
- **审批记录永久保留**：审批历史不可删除，确保审计追溯合规；
- **消息通知**：审批完成后（通过或拒绝），系统自动向提交人发送通知，包含审批结论及拒绝原因（如有）；
- **操作日志**：所有审批操作记录系统操作日志，包含操作人、操作时间、操作前后状态变更。
